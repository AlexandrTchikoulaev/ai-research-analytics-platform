Leia este manual do início ao fim antes de começares a fazer qualquer coisa.
Segue os passos pela ordem indicada. Não saltes nenhum.

O sistema é composto por:
  - Uma API e website local (FastAPI + HTML)
  - Uma base de dados PostgreSQL (dentro de Docker)
  - Um armazenamento de ficheiros MinIO (dentro de Docker)
  - Uma interface gráfica de gestão de bases de dados (pgAdmin, dentro de Docker)

Para funcionar precisas de instalar no computador:
  1. Git
  2. Python 3.11
  3. Docker Desktop
  4. Ollama (motor de inteligência artificial local)
  5. As bibliotecas Python do projeto (instaladas automaticamente)

Tempo estimado de instalação: 45 a 90 minutos (dependendo da velocidade da internet)


══════════════════════════════════════════════════════════════════════════════════
  PARTE 1 — INSTALAR O GIT E CLONAR O PROJETO
══════════════════════════════════════════════════════════════════════════════════

  Git é uma ferramenta que permite descarregar o código do projeto diretamente
  do GitHub (um repositório de código online). Vamos usá-la para obter todos os
  ficheiros do projeto de forma automática.

  PASSO 1 | Descarregar o instalador do Git

    Abrir o browser no site oficial do Git:  https://git-scm.com/download/win
    O download pode começar automáticamente.  
    
    Caso o download não tenha começado, carregue em "Click here to download".


  PASSO 2 | Instalar o Git

    Abrir o ficheiro descarregado (normalmente fica na pasta Downloads)

    Vai aparecendo uma série de janelas com opções. Em todas pode-se deixar
    as opções por defeito e clicar em "Next" (Seguinte). Não é preciso alterar nada.

    No final clicar em "Install" e "Finish" quando acabar de instalar.


  PASSO 3 | Verificar se o Git foi bem instalado.

    Carregar nas teclas  Windows + R  ao mesmo tempo escrever "cmd" e carregar Enter

    Escrever
      git --version
    e carregar Enter

    Deve aparecer algo como:
      git version 2.xx.x.windows.x

    Se aparecer essa mensagem, o Git está instalado corretamente.


  PASSO 4 — Escolher onde guardar o projeto.

    No ambiente de trabalho criar uma pasta e entitular como "Aplicação".


  PASSO 5 — Descarregar o projeto do GitHub.

    Abrir o terminal: Windows + R e escrever "cmd"

    Navegar para a pasta destinada á aplicação.
      
      Por exemplo, para ir para o Desktop:
        cd "C:\Users\optil\Desktop\Aplicação"

    Escreve o seguinte comando e prime Enter:
        git clone https://github.com/AlexandrTchikoulaev/Projeto-Final-Eu.git
    O Git vai descarregar todos os ficheiros do projeto. Aguardar até terminar.

    No final é criada uma pasta chamada "Projeto-Final-Eu" no local escolhido.

    Para confirmar que correu bem, escreve:
        dir Projeto-Final-Eu
      Deves ver ficheiros como "start.py", "requirements.txt", etc.

    ATENÇÃO: Guarda bem o caminho completo desta pasta. Vais precisar dele
    mais à frente (por exemplo: C:\Users\TeuNome\Desktop\Projeto-Final-Eu).


══════════════════════════════════════════════════════════════════════════════════
  PARTE 2 — INSTALAR O PYTHON 3.11
══════════════════════════════════════════════════════════════════════════════════

Python é a linguagem de programação usada neste projeto. É necessária a versão 3.11.

PASSO 6 | Descarregar o instalador do Python 3.11.

  Abrir o browser no site oficial do Python:
       https://www.python.org/downloads/release/python-3119/
  
  Nessa página, desce até à secção "Files" e clicar em "Windows installer (64-bit)" 
  — deverá ser um ficheiro com um nome parecido com "python-3.11.9-amd64.exe"
  Aguarda o download terminar.


PASSO 7 | Instalar o Python 3.11.

  Abrir o ficheiro "python-3.11.9-amd64.exe" que foi acabado de descarregar

  VAI APARECER UMA JANELA DE INSTALAÇÃO. MUITO IMPORTANTE:
     Na parte de baixo da janela, faz o check em:
       ☑  "Add python.exe to PATH"

  Clicar em "Install Now". Aguardar a instalação terminar. Clicar em "Close".

  OPCIONAL mas recomendado: se aparecer um botão "Disable path length limit",
     clicar nele. Resolve problemas com caminhos de ficheiros muito longos.


PASSO 8 | Verificar se o Python foi bem instalado.

  Carregar nas teclas  Windows + R  ao mesmo tempo. Escrever "cmd". Carregar enter.

  Na janela preta, escrever:
       python --version
  e carregar Enter

  Deve aparecer algo como:
       Python 3.11.9


══════════════════════════════════════════════════════════════════════════════════
  PARTE 3 — INSTALAR O DOCKER DESKTOP
══════════════════════════════════════════════════════════════════════════════════

Docker é uma ferramenta que cria "contentores" — ambientes isolados onde correm
a base de dados e o armazenamento de ficheiros. É como ter pequenos servidores
a correr dentro do teu computador, sem precisar de os instalar manualmente.

PASSO 9 | Verificar os requisitos do Docker Desktop.

  O Docker Desktop requer (opcional):
    - Windows 10 versão 1903 ou superior (ou Windows 11)
    - Pelo menos 4 GB de RAM (recomendado 8 GB)
    - Virtualização ativada na BIOS (na maioria dos computadores modernos já está)

  Para verificar a versão do Windows:
    a) Carrega  Windows + R
    b) Escreve "winver" e prime Enter
    c) Verifica a versão que aparece


PASSO 10 | Descarregar o Docker Desktop.

  Abrir o browser no site oficial do Docker: https://www.docker.com/products/docker-desktop/
  Clicar no botão de download para Windows (deve ser um ficheiro chamado "Docker Desktop Installer.exe")
  Aguardar o download


PASSO 11 | Instalar o Docker Desktop.

  Abrir o ficheiro "Docker Desktop Installer.exe"

  Na janela de instalação, deixa as opções por defeito selecionadas:
    ☑  Use WSL 2 instead of Hyper-V (recomendado)
    Clica em "Ok"

  No final, clicar em "Close and restart" para reiniciar o computador.


PASSO 12 | Configurar o Docker Desktop após reinício.

  Após o reinício, o Docker Desktop pode abrir automaticamente. Se não abrir,
  procurar "Docker Desktop" no menu Iniciar e abri-lo.

  É possível que apareça uma mensagem sobre instalar o "WSL 2 kernel update".
  Se aparecer, clica no link e instala a atualização. Depois fecha e reabre
  o Docker Desktop.

  Aguarda até o ícone do Docker (uma baleia azul) aparecer na barra de tarefas
     (zona do relógio, no canto inferior direito).

  O ícone pode ficar animado (a baleia com contentores a mover) durante o
     arranque. Espera até ficar estático — isso significa que o Docker está pronto.


PASSO 13 — Verificar se o Docker foi bem instalado.

  Abrir uma nova janela do terminal:
       Windows + R → escreve "cmd" → prime Enter

  Escrever:
       docker --version
  e prime Enter

  Deve aparecer algo como:
       Docker version 27.x.x, build xxxxxxx

  Escreve também:
       docker compose version
     e prime Enter

  Deve aparecer algo como:
       Docker Compose version v2.x.x

  Se ambos os comandos funcionaram, o Docker está instalado corretamente.


══════════════════════════════════════════════════════════════════════════════════
  PARTE 4 — INSTALAR AS BIBLIOTECAS PYTHON DO PROJETO
══════════════════════════════════════════════════════════════════════════════════

O projeto usa várias bibliotecas Python especializadas que precisam de ser
instaladas. Estão listadas no ficheiro "requirements.txt" dentro da pasta
do projeto.

PASSO 14 | Abrir o terminal na pasta do projeto.

  Abre o terminal normalmente (Windows + R → cmd → Enter)
  Escreve o comando do caminho: cd C:\Users\TeuNome\Desktop\Projeto-Final-Eu
  e carregar Enter

  Para confirmar que estás na pasta certa, escreve:
       dir
  e prime Enter. Deves ver ficheiros como "start.py", "requirements.txt", etc.


PASSO 15 — Instalar as bibliotecas Python.

  Com o terminal aberto NA PASTA DO PROJETO, escreve o seguinte comando e
  prime Enter:
       pip install -r requirements.txt


══════════════════════════════════════════════════════════════════════════════════
  PARTE 5 — INSTALAR O OLLAMA E OS MODELOS DE IA
══════════════════════════════════════════════════════════════════════════════════

O Ollama é um motor que corre modelos de inteligência artificial localmente no
teu computador, sem necessidade de internet nem de serviços externos pagos.
O sistema usa dois modelos: "qwen2.5:7b" (para gerar código e responder a
perguntas) e "mxbai-embed-large" (para pesquisa semântica nos relatórios).

ATENÇÃO: Os modelos ocupam espaço em disco. Certifica-te de que tens pelo
menos 10 GB livres antes de começar este passo.


PASSO 16 | Descarregar o instalador do Ollama.

  Abrir o browser no site oficial do Ollama:  https://ollama.com/download
  Clicar em "Download for Windows".
  Aguardar o download terminar (ficheiro chamado "OllamaSetup.exe").


PASSO 17 | Instalar o Ollama.

  Abrir o ficheiro "OllamaSetup.exe" que foi acabado de descarregar.

  A instalação é automática — não aparece nenhum assistente com opções.
  O Ollama instala-se e fica a correr em segundo plano.

  Após a instalação, aparece o ícone do Ollama (uma chama laranja) na barra
  de tarefas (zona do relógio, canto inferior direito).


PASSO 18 | Verificar se o Ollama foi bem instalado.

  Abrir o terminal: Windows + R → escrever "cmd" → carregar Enter

  Escrever:
       ollama --version
  e carregar Enter

  Deve aparecer algo como:
       ollama version 0.x.x

  Se aparecer essa mensagem, o Ollama está instalado corretamente.


PASSO 19 | Descarregar os modelos de IA.

  Este passo descarrega os dois modelos necessários. Pode demorar bastante
  tempo (20 a 60 minutos) dependendo da velocidade da internet.
  Os modelos são descarregados uma única vez e ficam guardados no computador.

  Com o terminal aberto, escreve o primeiro comando e prime Enter:
       ollama pull qwen2.5:7b

  Aguarda até aparecer "success" ou "pull complete". Depois escreve:
       ollama pull mxbai-embed-large

  Aguarda novamente até aparecer "success" ou "pull complete".


PASSO 20 | Verificar se os modelos foram descarregados.

  No terminal, escreve:
       ollama list

  Deves ver os dois modelos na lista, por exemplo:
       NAME                     ID              SIZE
       qwen2.5:7b               ...             ~4.7 GB
       mxbai-embed-large        ...             ~670 MB

  Se ambos aparecerem, os modelos estão prontos a usar.


══════════════════════════════════════════════════════════════════════════════════
  PARTE 6 — CONFIGURAÇÃO INICIAL DO SISTEMA (SÓ FAZER UMA VEZ)
══════════════════════════════════════════════════════════════════════════════════

Este passo cria as bases de dados, configura o armazenamento de ficheiros e
prepara tudo para a primeira utilização. SÓ PRECISAS DE FAZER ISTO UMA VEZ.

PASSO 21 — Garantir que o Docker Desktop está a correr.

  Olha para a barra de tarefas (zona do relógio, canto inferior direito).
  Deve aparecer o ícone do Docker (baleia azul).
  Se não aparecer, procura "Docker Desktop" no menu Iniciar e abre-o.
  Aguarda até o ícone ficar estático (não animado) — o Docker está pronto.


PASSO 22 — Executar o script de configuração inicial.

  Abre o terminal NA PASTA DO PROJETO (como explicado no Passo 14)

  Escreve o seguinte comando e prime Enter:
      python Codes/Setup/setup.py 

  O script vai fazer automaticamente (pode demorar 5-15 minutos):
       - Verificar se o Docker está disponível
       - Iniciar os contentores (base de dados, pgAdmin, MinIO)
       - Aguardar que os serviços fiquem prontos
       - Criar as bases de dados necessárias
       - Criar as tabelas
       - Executar o ETL (processar os dados)

  Vais ver muitas mensagens no terminal. É normal. Mensagens com "OK" ou
     "concluído" são boas. Mensagens com "ERRO" indicam problemas.

  No final, se correr tudo bem, aparece:
       ╔══════════════════════════════════════════════════════════╗
       ║         SETUP CONCLUÍDO COM SUCESSO                       ║
       ╚══════════════════════════════════════════════════════════╝

  ATENÇÃO: Se aparecer um erro durante este passo, não entres em pânico.
  Lê a mensagem de erro com atenção e verifica:
    - O Docker Desktop está a correr? (ícone na barra de tarefas)
    - Estás na pasta certa? (usa "dir" para confirmar)
    - As bibliotecas Python foram instaladas? (repete o Passo 15)


══════════════════════════════════════════════════════════════════════════════════
  PARTE 7 — CRIAR O ÍCONE NO AMBIENTE DE TRABALHO (SÓ FAZER UMA VEZ)
══════════════════════════════════════════════════════════════════════════════════

Este passo cria um ícone no ambiente de trabalho para que no futuro baste fazer
duplo clique para abrir a aplicação. SÓ PRECISAS DE FAZER ISTO UMA VEZ.

PASSO 23 — Criar o ícone no ambiente de trabalho.

  Abre o terminal NA PASTA DO PROJETO (como explicado no Passo 14)

  Escreve o seguinte comando e prime Enter:
      python App/criar_atalho.py

  Deve aparecer no terminal:
       Atalho criado: C:\Users\TeuNome\Desktop\OP Report Manager.lnk
       Podes fechar isto. O ícone está no ambiente de trabalho.

  Vai ao ambiente de trabalho — deves ver um novo ícone chamado
     "OP Report Manager".

  A partir de agora, para abrir a aplicação basta fazer duplo clique nesse ícone.
  Não precisas mais de abrir o terminal para isso.

  RESOLUÇÃO DE PROBLEMAS COMUNS
══════════════════════════════════════════════════════════════════════════════════

PROBLEMA: "python não é reconhecido como um comando interno ou externo"
SOLUÇÃO:  O Python não foi adicionado ao PATH. Desinstala o Python (via
          "Adicionar ou Remover Programas") e repete o PASSO 3, desta vez
          garantindo que marcas a opção "Add python.exe to PATH".

----------

PROBLEMA: "docker não é reconhecido como um comando interno ou externo"
SOLUÇÃO:  O Docker Desktop não está a correr ou não foi instalado corretamente.
          Abre o Docker Desktop pelo menu Iniciar. Se não aparecer, reinstala-o
          (PASSO 6 e 7).

----------

PROBLEMA: O Docker Desktop não inicia / fica sempre "a carregar"
SOLUÇÃO A: Reinicia o computador e tenta novamente.
SOLUÇÃO B: Pode ser necessário ativar a virtualização na BIOS. Contacta o aluno
           ou o suporte de informática da instituição.
SOLUÇÃO C: Desinstala e reinstala o Docker Desktop.

----------

PROBLEMA: Erro "port is already in use" ou "porta já em uso"
SOLUÇÃO:  Algum serviço no computador está a usar a mesma porta. Reinicia o
          computador e tenta novamente. Se persistir, contacta o aluno.

----------

PROBLEMA: O website abre mas não carrega dados / aparece em branco
SOLUÇÃO:  O setup inicial pode não ter corrido corretamente. Tenta correr
          novamente:  python Codes/Setup/setup.py

----------

PROBLEMA: O chat de dados ou de relatórios não responde / dá erro de ligação
SOLUÇÃO:  O Ollama não está a correr. Procura "Ollama" no menu Iniciar e abre-o.
          Aguarda até aparecer o ícone da chama laranja na barra de tarefas.
          Se o ícone já estiver presente, verifica se os modelos estão instalados:
            ollama list
          Se "qwen2.5:7b" ou "mxbai-embed-large" não aparecerem, corre
          novamente os comandos do PASSO 19.

----------

PROBLEMA: "ollama não é reconhecido como um comando interno ou externo"
SOLUÇÃO:  O Ollama não foi instalado ou precisa de reiniciar o terminal.
          Fecha todas as janelas do terminal, abre uma nova e tenta outra vez.
          Se persistir, reinstala o Ollama (PASSO 16 e 17).

----------

PROBLEMA: "No module named 'fastapi'" ou outro módulo em falta
SOLUÇÃO:  As bibliotecas não foram instaladas. Repete o PASSO 11.

----------

PROBLEMA: O sistema fica lento ou o computador fica pesado
EXPLICAÇÃO: O Docker está a correr vários serviços em simultâneo, o que consome
            memória RAM. É normal alguma lentidão. Quando não estiveres a usar
            o sistema, para-o sempre (PASSO 16).

----------

PROBLEMA: Perdeste ou não consegues encontrar a pasta do projeto
SOLUÇÃO:  Usa o Windows Search (tecla Windows, escreve "start.py") para
          encontrar o ficheiro e descobrir onde está a pasta.

     → Fica um ícone "OP Report Manager" no ambiente de trabalho
