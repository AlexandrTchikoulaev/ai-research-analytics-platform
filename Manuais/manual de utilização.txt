══════════════════════════════════════════════════════════════════════════════════
  PARTE 1 — INICIAR O SISTEMA
══════════════════════════════════════════════════════════════════════════════════

PASSO 1 — Iniciar o sistema.

  a) Garante que o Docker Desktop está a correr.
     Olha para a barra de tarefas (zona do relógio, canto inferior direito).
     Deve aparecer o ícone do Docker (baleia azul, estático e não animado).
     Se não aparecer, procura "Docker Desktop" no menu Iniciar e abre-o.
     Aguarda até o ícone ficar estático — só então continua.

  b) Faz duplo clique no ícone "OP Report Manager" no ambiente de trabalho.
     (Se não tiveres o ícone, abre o terminal na pasta do projeto e escreve:
       python criar_atalho.py
     Só precisas de fazer isso uma vez.)

  d) O sistema demora cerca de 1-2 minutos a arrancar completamente.
     Durante esse tempo aparece um ícone na barra de tarefas (zona do relógio).

  e) Quando o sistema estiver pronto, aparece uma notificação:
       "Sistema pronto! — OP Report Manager"
     E o browser abre automaticamente com o website.


══════════════════════════════════════════════════════════════════════════════════
  PARTE 2 — UTILIZAR O SISTEMA
══════════════════════════════════════════════════════════════════════════════════

PASSO 2 — Navegar no sistema.

  O ícone na barra de tarefas (canto inferior direito, junto ao relógio) tem
  um menu que aparece com o botão direito do rato:

    • "Abrir"        → abre a janela principal do sistema
    • "Parar Sistema" → encerra tudo de forma segura (usa sempre isto para fechar!)

  Serviços disponíveis:
    - Website/API:   http://localhost:8000
    - pgAdmin:       http://localhost:5051
                     (utilizador: admin_projeto@teste.com  senha: admin123)
    - MinIO Console: http://localhost:9003
                     (utilizador: admin  senha: admin123)


══════════════════════════════════════════════════════════════════════════════════
  PARTE 3 — PARAR O SISTEMA
══════════════════════════════════════════════════════════════════════════════════

PASSO 3 — Parar o sistema.

  IMPORTANTE: Nunca feches o computador ou a janela sem parar o sistema primeiro!
  Usa sempre a opção "Parar Sistema" do ícone na barra de tarefas.

  Alternativamente, abre o terminal na pasta do projeto e escreve:
       python stop.py


══════════════════════════════════════════════════════════════════════════════════
  PARTE 4 — CONFIGURAR CHAVE API (SE NECESSÁRIO PARA O CHAT INTELIGENTE)
══════════════════════════════════════════════════════════════════════════════════

O sistema tem uma funcionalidade de chat inteligente que usa inteligência
artificial (LangChain). Para essa funcionalidade funcionar, pode ser necessário
configurar uma chave de API.

PASSO 4 — Configurar a chave de API (se pedido).

  Se ao usar o chat aparecer um erro sobre "API key" ou "authentication":

  a) O aluno deve fornecer a chave de API necessária (OpenAI ou outra).

  b) Para configurar, cria uma variável de ambiente no Windows:
       - Carrega Windows + R, escreve "sysdm.cpl" e prime Enter
       - Clica no separador "Avançadas"
       - Clica em "Variáveis de Ambiente"
       - Em "Variáveis do sistema", clica em "Novo"
       - Nome: OPENAI_API_KEY  (ou o nome que o aluno indicar)
       - Valor: (a chave fornecida pelo aluno)
       - Clica OK em todas as janelas

  c) Fecha e reabre o terminal para as alterações terem efeito.
